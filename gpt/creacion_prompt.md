# Tips para crear un buen prompt

### Contexto

Se entiende como contexto al entorno, la información o las características que rodean o influyen en la generación de contenido por parte del modelo, por ejemplo, el contexto puede incluir texto o las palabras que lo rodean. El modelo utiliza este contexto para generar una respuesta coherente o relevante.

### Construcción del prompt

El prompt o tambieón conocido como entrada o estímulo, es la solicitud o la señal que se envía al modelo generativo para que produzca una respuesta. El prompt puede ser una pregunta, una frase incompleta, una instrucción o cualquier otra forma de entrada que guíe al modelo en la generación del contenido deseado.  
El prompt seuele ser más específico y directo que el contexto, ya que esta diseñado para influir directamente en el tipo de salida que el modelo generará.  
Para obtener un buen prompt es vital construir un buen prompt, el desarrollador debe:

1. Especificar de forma clara y ordenada lo que se desea
2. Añadir cualquier información que el developer considere relevante para la realización de la tarea
3. Iterar tantas veces sea necesario hasta conseguir la salida deseada

### Alucinaciones

las "alucinaciones" se refieren a resultados inesperados incoherentes o poco realistas producidos por un modelo generativo. Las alucinaciones pueden ocurrir por diversas razones, como la falta de contexto adecuado, la ambigüedad en la solicitud de generación, la presencia de sesgos en los datos de entrenamiento del modelo o limitaciones inherentes en la capacidad del modelo para comprender y producir contenido coherente. Es importante tener en cuenta que las alucinaciones son un desafío común en el desarrollo y el uso de modelos generativos, especialmente en aquellos que son muy avanzados y complejos, como los modelos de lenguaje basados en inteligencia artificial como GPT (Generative Pre-trained Transformer). Es responsabilidad de los desarrolladores de verificar que las sugerencias aceptadas cumplen realmente con la funcionalidad requerida.
